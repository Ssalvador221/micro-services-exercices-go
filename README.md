# Microservices Architecture Journey (Go)

This repository demonstrates the evolution of a backend system using different architectural patterns:

## 🚀 Versions

- v1: Monolith (baseline)
- v2: Clean Architecture
- v3: Hexagonal Architecture
- v4: Microservices (gRPC + Docker + Kubernetes)

## 🎯 Goal

Transform a simple inventory system into a scalable microservices architecture.

## 🧠 What you'll learn

- Go project structure
- Clean Architecture
- DDD basics
- Hexagonal Architecture
- gRPC communication
- Kubernetes deployment
